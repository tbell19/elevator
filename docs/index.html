Elevator Project

by Trent Bell

The Elevator I Chose

I chose the elevators closest to the enterance of the Texas Tech Library

The current design is a little busy and has a confusing button layout.

There are only four floors so there is no reason why the buttons cannot be stacked vertically.

Currently it is not intuitive to find the button for the floor that you are trying to go to with the buttons staggered left and right.

The elevator has lots of information but it is displayed in a way which can be overwhelming when trying to make a quick glance

There is very little seperation the buttons which are used often and the buttons which are only used in emergency situations

About this Elevator

This elevator is commonly used for

  • Taking students between floors
  • Transferring books and equipment between floors
  • Providing information about the library

Uncommon uses for this elevator

  • Emergency use by firefighters
  • Calling for help in the event of a malfunction

A typical interaction with this elevator would be a person entering, pressing the button for their selected floor and waiting for it to arrive.

The Redesign

Original Design

  • The original design has a list of the things on each floor surrounding the buttons which is overwhelmin
  • The elevator uses the intuitive nature of pressing the button next to the floor name to support the user figuring out how to use it on their own; however, the spacing and layout of buttons could make this confusing.
  • The elevator gives feedback via beeps and the tactile feel of the buttons.
  • Some common mistakes that can be made while designing an elevator is not putting the buttons in an order that makes sense, this elevator I believe has that mistake.

My Design

  • My design features a less cluttered screen which shows information for only the current floor.
  • My elevator also uses the intuitive nature of pressing buttons which are labeled with their corresponding floor, however my buttons are layed out with the floor buttons stacked vertically on the left side and with extra buttons which are not used often on the right.
  • My elevator gives feedback via beeps and the tactile feel of the buttons, but also features additional information on the screen.
  • I believe my elevator avoids many common design mistakes and provides an intuitive experience for users.
